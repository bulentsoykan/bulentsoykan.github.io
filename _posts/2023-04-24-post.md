# Non-Linear Programming

## Introduction

Nonlinear programming (NLP) is the process of solving an optimization problem where some of the constraints or the objective function are nonlinear. An optimization problem is one of calculation of the extrema (maxima, minima or stationary points) of an objective function over a set of unknown real variables and conditional to the satisfaction of a system of equalities and inequalities, collectively termed constraints.

- **Linear Programming (LP)**: The objective function and the constraints are linear functions of the decision variables.
- **Nonlinear Programming (NLP)**: The objective function and/or the constraints are nonlinear functions of the decision variables.

- **Convex Programming (CP)**: The objective function and the constraints are convex functions of the decision variables.
- **Non-Convex Programming (NCP)**: The objective function and/or the constraints are non-convex functions of the decision variables.

- **Quadratic Programming (QP)**: The objective function is a quadratic function of the decision variables.
- **Quadratically Constrained Programming (QCP)**: The constraints are quadratic functions of the decision variables.

# Non-Linear Models in Mathematical Programming

Non-linear models have diverse applications in mathematical programming. They come into play when linear programming's assumptions, such as constant returns to scale, proportional resource usage, and additive resource consumption, are not met. This can result in increasing or decreasing returns to scale, leading to non-linear terms in the objective function.

## Sources of Non-Linearity

Non-linearity can emerge from various sources:

- **Returns to Scale**: Increasing returns to scale introduce non-linear terms like $2x1^2$ in the objective function.
- **Price Elasticity**: Unit selling price impacting profit introduces non-linearity, often approximated by a linear function of quantity.
- **Constraints**: Non-linearities can appear in constraints, such as those in blending problems or geometric programming models.
- **Geometric Programming**: Engineering problems often involve polynomial constraints, characteristic of geometric programming models.

## Convex and Non-Convex Programming

Non-linear programming can be categorized into convex and non-convex programming:

- **Convexity**: A region is convex if the line connecting any two points in the region lies entirely within it. A function is convex if the set of points (x, y) where y ≥ f(x) forms a convex region. A mathematical programming model is convex if it minimizes a convex function over a convex feasible region.
- **Linear Programming**: Linear programming is a subset of convex programming where both the objective function and constraints are linear. Convex functions have a unique global minimum, simplifying optimization.
- **Local and Global Optima**: Non-convex functions can have multiple local minima. Algorithms may find local optima, but global optima can be elusive, making non-convex model solving more complex.

## Separable Programming and Piecewise Linear Approximations

One approach to non-linear models is separable programming:

1. Make piecewise linear approximations to non-linear functions.
2. Introduce new variables (e.g., λi or δi) to represent weights attached to the vertices of piecewise lines.
3. Impose stipulations to ensure solutions lie on the piecewise lines.

The δ-formulation is an alternative to the λ-formulation for modeling piecewise linear approximations, often offering a more relaxed relaxation in linear programming.

## Handling Non-Convexity

Solving non-convex models requires advanced techniques. While separable programming provides local optima, guaranteeing a global optimum often demands methods like integer programming. The separable programming extension of the simplex algorithm helps mitigate local optima issues by restricting certain values, but challenges remain.

In conclusion, non-linear models enrich mathematical programming by addressing scenarios beyond linear programming's scope. Their applications span various fields, making them an essential tool for optimization in real-world problems.
